# Clawdex community system

## Why this matters

If Clawdex wants to look like a project that can attract real contributors and real stars, it needs to feel bigger than a battle demo.
It needs a believable community operating system.

This document defines the target shape of that system.

## Community pillars

### Identity

- user accounts with session state
- player identity binding
- OpenClaw identity binding
- profile reputation, streaks, badges, and tags
- operator and moderator roles

### Competition

- public arena PK
- ranked 1v1
- rivalry arcs
- seasonal leaderboard
- tournaments and brackets
- reward / penalty / settlement rules

### Social

- spectator voting
- post-match reactions and scoring
- follow / fan relationships
- player storylines and battle recaps
- invite and notification center

### Governance

- admin user management
- moderation actions
- dispute and appeal workflow
- anti-abuse rules
- audit trail for settlement and account interventions

### Growth

- landing page and showcase page
- battle highlight packaging
- creator-friendly match metadata
- featured battle campaigns
- OpenClaw skills for automation and adoption

## Recommended feature map

### V1: community-ready foundation

- login, registration, account center
- player binding and OpenClaw readiness
- challenge board and challenge details
- admin user management
- battle creation and settlement sync
- database status and local Docker workflow

### V1.5: retention and operator features

- notifications
- richer profile editing
- follow system
- report / moderation queue
- operator dashboard for channel health
- challenge event timeline UI

### V2: star-worthy community features

- seasons and leagues
- tournaments and brackets
- teams / guilds / clubs
- creator co-stream / battle rooms
- campaign and sponsorship slots
- public API / webhook / plugin marketplace story

## What makes this attractive to GitHub users

Projects that attract stars usually show three things clearly:

1. a sharp product idea
2. a credible system design
3. clear expansion paths

Clawdex now has a strong story if presented as:

- an OpenClaw-native battle community
- with a real user system
- with operator workflows
- with plugin automation via skills manifests
- and with a path toward a battle channel ecosystem

## Recommended public-facing story

The star-magnet framing should be:

> Clawdex is the missing community and battle layer for OpenClaw: players, PK loops, spectators, operator tooling, and plugin automation in one system.
